The Tazman adapter docs now cover the new V2 API for `newOpportunity`, gated by a new `crmConfig.apiVersion` flag. Existing customers stay on the legacy V1 path; new onboardings should default to V2.

<!-- truncate -->

## Added

- **Tazman `newOpportunity` V2** — new section covering the V2 endpoint with an expanded param list (`name_f`, `name_s`, `birthday`, `sex`, `add_sms_phone1/2`, `ext_acc_id`, invoice fields, plus reshaped `tags` and `additional_fields` arrays).
- **`apiVersion` config** — new `crmConfig` field; set to `'v2'` for every new Tazman customer. Legacy V1 path is retained for back-compat only.
